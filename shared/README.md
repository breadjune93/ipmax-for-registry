# shared — 언어 중립 공통 정의

> 모든 플랫폼(서버·라우터·Windows·Android·macOS·iOS)이 **똑같이 해석해야 할** 상수·모델·규약을 이 폴더에 둔다.
> 여기의 변경은 **파급 범위가 크다**. 반드시 ADR과 함께 변경한다.

## 무엇이 들어가나

- **`models/`** — API 스펙으로 다 표현되지 않는 **도메인 정의**(enum 값의 의미, 상태 전이 규칙 등)
- **`constants/`** — 에러 코드 카탈로그, 단위 변환 기준, 정책 임계값(예: Heartbeat 허용 누락 수)
- **`schemas/`** — 이벤트 페이로드 JSON 스키마(Kafka 등)

## 무엇이 들어가지 않나

- 특정 언어 전용 코드 (Java/Kotlin/Swift/C# 구현) → 각 플랫폼 폴더로
- REST API 스펙 → `../docs/api-contracts/openapi.yaml` 이 진실
- 데이터베이스 스키마 → `server/` 의 마이그레이션이 진실

## 소비 방식

각 플랫폼은 빌드 시점에 이 폴더를 참조해서 상수/enum을 **생성**하거나 정적 체크에 사용.
예) 에러 코드 카탈로그(JSON/YAML) → 각 언어의 enum으로 코드 생성.

## 변경 절차

1. 변경 제안 → ADR 작성(`../docs/business/decisions/`)
2. 이 폴더 업데이트
3. 영향 받는 플랫폼에 이슈 생성, 전 플랫폼 동기화 확인 후 배포
