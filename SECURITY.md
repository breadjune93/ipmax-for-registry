# SECURITY

IPMax는 사용자 네트워크와 금전이 오가는 서비스이므로 **보안 이슈는 최우선**으로 처리합니다.

## 취약점 보고

- **공개 이슈로 등록하지 마세요.**
- 비공개 채널: `security@ipmax.example.com` (실제 주소로 교체)
- 가능하면 PGP 키로 암호화 (키는 회사 보안 가이드 참고)

## 보고 시 포함할 내용

- 영향 범위(서버/라우터/클라이언트)
- 재현 단계
- 예상 영향(데이터 유출, 권한 상승, 서비스 중단 등)
- 제안 완화책

## 내부 원칙

- 비밀값(.env, keystore, WG private key) **리포지토리 커밋 금지**
- 모든 관리자 액션은 **MFA + 감사 로그**
- 의존성 취약점은 CI에서 자동 스캔(추후 도입) — 우선순위 HIGH 이상은 72시간 내 대응
- 사용자 IP·세션 메타는 **최소 보존** + 마스킹 정책 준수
